# 商家/商品数据约定

生成新数据时对齐这些约定，保持和现有 14 家商家的风格一致。

## 文件与字段

- 每个商家一个文件：`src/data/restaurants/<id>.json`，`id` 用小写英文短拼音/缩写（如 `haidilao`、`kfc`）。
- 必须包含 `order: number` 字段，取 `src/data/restaurants/` 下所有现有文件里最大 `order` 值 + 1（新商家排在最后）。
- 完整字段结构见 `src/data/types.ts` 的 `Restaurant` / `MenuItem` interface。
- 新商家生成数据阶段先不填 `bannerImage`/`image`/`imagePrompt`/`bannerImagePrompt`（等第 3、4 步生成 prompt 和图片时再回填）；`seriesStyle` 在第 3 步生成。

## 商品 id 前缀规则

商品 `id` = `<商家简码>_<商品名缩写拼音>`，如 `haidilao` 的商品用 `hdl_xxx`、`starbucks` 用 `sbk_xxx`。简码取商家名字的声母缩写或英文缩写，3~4 个字母为宜，同一商家内部保持统一前缀（不要一半用拼音缩写一半用英文）。目的是让所有商品 id 全局唯一、且一眼能看出属于哪个商家。

## 商品数量与分类

- 每个商家至少 30 个商品（含大于 30，参考现有 14 家都是 30~32 个）。
- `menuCategories` 至少 5~8 个（如"热销"、按菜系拆的具体分类、"饮品"、"小吃"、"套餐"等），第一个通常是"热销"。
- `menu` 里每个商品的 `menuCategory` 必须是 `menuCategories` 里已存在的值，否则前端 tab 不会显示。
- 每个商家挑 2~4 个商品打 `popular: true`（通常放在"热销"分类下）。

## 价格 / 卡路里合理区间（参考现有商家定价）

| 品类 | 单品价格区间 | 卡路里区间 |
|---|---|---|
| 快餐正餐（饭/面/堡） | ¥12~35 | 400~700 |
| 小吃/配菜 | ¥3~20 | 30~450 |
| 饮品（奶茶/咖啡/汤饮） | ¥5~40 | 10~420 |
| 火锅/烧烤类单品食材 | ¥4~35 | 20~450 |
| 甜点/烘焙单品 | ¥5~50 | 150~550 |
| 套餐/拼盘（多份组合） | ¥35~160 | 850~3000 |

新商家如果是新菜系，可参考同类现有商家（如新增另一家咖啡店可参考 `starbucks.json` 定价）微调，不要凭空编造偏离现有价格带的数字。

## `Category`（顶层分类 tab）

现有值：`全部` `中式快餐` `汉堡炸鸡` `日料韩料` `奶茶饮品` `小吃零食` `火锅` `披萨` `咖啡烘焙` `甜点烘焙`。

新商家如果不属于任何现有分类，需要：
1. 在 `src/data/types.ts` 的 `Category` 联合类型里加上新分类字符串。
2. 在 `src/data/restaurants.ts` 聚合器里的 `CATEGORIES` 数组里追加同一个字符串（顺序即首页 tab 显示顺序）。
