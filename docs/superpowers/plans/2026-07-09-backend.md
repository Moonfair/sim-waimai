# sim-waimai 后端功能实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为纯前端的外卖模拟应用引入真实 Node 后端：账号系统（JWT）、持久化订单（快照+游标分页）、用户注册商家与管理后台、评价/收藏/推荐，图片走腾讯云 COS（本地回退）。

**Architecture:** npm workspaces 单仓：根为 Vite React 客户端，新增 `shared/`（共享类型/DTO/zod schema）与 `server/`（Hono + Drizzle + PostgreSQL 16）。开发时 Vite 代理 `/api` 到 :3001，同源无 CORS。商家数据从构建期 JSON 迁入数据库（JSON 保留为种子）。

**Tech Stack:** Hono、@hono/node-server、@hono/zod-validator、drizzle-orm、drizzle-kit、pg、bcryptjs、cos-nodejs-sdk-v5、zod、tsx、vitest、concurrently。

**设计文档：** `docs/superpowers/specs/2026-07-09-backend-design.md`（Schema、API 表、选型理由均以其为准）

## Global Constraints

- UI 全中文，沿用现有 Tailwind（`dark:` 变体、orange-500 主色调）与 Context 模式。
- TS strict + `verbatimModuleSyntax`：shared 里保持 type-only import。
- 金额：DB 存分（integer `*_fen`），API 响应给前端元（number）；JSONB 快照存前端形状（元）。
- 服务端永远重新计价，不信任客户端价格。
- 现有 6 条路由的既有行为不回归（购物车、下单动画、骑手倒计时）。
- 每个任务结束跑通验证再 commit；commit message 末尾带 Co-Authored-By Claude 行。

**Interfaces:**
